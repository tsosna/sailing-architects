Expert review

Aktualny kierunek jest dobry, ale zbyt cichy tam, gdzie ma sprzedawać zaufanie.

6.7 /10 · dobre fundamenty, wymagane korekty hierarchii

Styl premium jest spójny z marką, lecz trzy elementy krytyczne dla konwersji nie mają wystarczającego priorytetu: człowiek do kontaktu, etykiety orientacyjne oraz modal logowania w procesie płatności.

Filozofia
7.5

Hierarchia
5.8

CTA jest widoczne, lecz kontakt i logowanie nie tworzą jasnych punktów wejścia. Eyebrow przy opacity 0.6 bywa dekoracją zamiast sygnałem.

Craft
7.0

System odstępów i ostre krawędzie są stabilne. Do poprawy: kontrast małych napisów oraz warstwowanie panelu Clerk.

Funkcja
6.2

Użytkownik może zarezerwować koję, ale w momencie wątpliwości musi szukać kontaktu, a logowanie wygląda jak część tła.

Oryginalność
6.8

Jest potencjał na własny język: manifest, kapitanat, sygnał portowy. Obecnie część UI jest jeszcze zbyt typowa.

Fix 01
Kontakt jako sygnał zaufania

Numer i e-mail powinny wejść do hero lub pierwszego kroku booking flow jako element pierwszego rzędu, nie dopiero FAQ/stopka.

Fix 02
Eyebrow bez opacity 0.6

Dla małego tekstu użyć brass-light, pełnej nieprzezroczystości, większej wagi albo jasnego mikro-tła.

Fix 03
Clerk jako osobny panel

Logowanie musi dostać własną warstwę: border-top brass, navy-deep tło, inputy navy-light i cień tylko jako funkcjonalna separacja.
